# 🤖 Kiota Explained Simply

## 🎯 What is Kiota?

Kiota is a Microsoft tool that **automatically generates C# code** from an OpenAPI (Swagger) file.

**Analogy:** It's like having an assistant who reads an API's documentation and writes all the C# code needed to call that API.

---

## 📝 Step by Step: How It Works

### 1️⃣ You have an API (ServiceB)

```csharp
// ServiceB/Program.cs
app.MapGet("/api/products", () =>
{
    return new[]
    {
        new { Id = 1, Name = "Laptop", Price = 999.99 },
        new { Id = 2, Name = "Mouse", Price = 29.99 }
    };
});

app.MapGet("/api/products/{id}", (int id) =>
{
    return new { Id = id, Name = "Laptop", Price = 999.99 };
});
```

---

### 2️⃣ You create an OpenAPI file describing the API

```json
// ServiceB/openapi.json
{
  "openapi": "3.0.1",
  "info": {
    "title": "ServiceB - Products API",
    "version": "1.0.0"
  },
  "paths": {
    "/api/products": {
      "get": {
        "operationId": "getProducts",
        "responses": {
          "200": {
            "description": "Success",
            "content": {
              "application/json": {
                "schema": {
                  "type": "array",
                  "items": { "$ref": "#/components/schemas/Product" }
                }
              }
            }
          }
        }
      }
    },
    "/api/products/{id}": {
      "get": {
        "operationId": "getProductById",
        "parameters": [
          { "name": "id", "in": "path", "required": true, "schema": { "type": "integer" } }
        ],
        "responses": {
          "200": {
            "content": {
              "application/json": {
                "schema": { "$ref": "#/components/schemas/Product" }
              }
            }
          }
        }
      }
    }
  },
  "components": {
    "schemas": {
      "Product": {
        "type": "object",
        "properties": {
          "id": { "type": "integer" },
          "name": { "type": "string" },
          "price": { "type": "number" }
        }
      }
    }
  }
}
```

**This file acts as a "contract" that describes:**
- Which endpoints exist? `/api/products`, `/api/products/{id}`
- What parameters do they accept? `id` (integer)
- What do they return? Array of `Product` or a single `Product`
- Structure of `Product`? `id`, `name`, `price`

---

### 3️⃣ You run the Kiota command

```bash
cd ServiceA
kiota generate \
  -l CSharp \
  -d ../ServiceB/openapi.json \
  -o ./Generated/ServiceBClient \
  -n ServiceA.Generated.ServiceBClient
```

**What this command does:**
- `-l CSharp` → Generates code in C#
- `-d ../ServiceB/openapi.json` → Reads the API contract
- `-o ./Generated/ServiceBClient` → Saves the generated files here
- `-n ServiceA.Generated.ServiceBClient` → Namespace for the generated code

---

### 4️⃣ Kiota automatically generates MULTIPLE C# files

```
ServiceA/Generated/ServiceBClient/
├── ApiClient.cs                           # Main client
├── Api/
│   ├── ApiRequestBuilder.cs              # Builder for /api
│   └── Products/
│       ├── ProductsRequestBuilder.cs     # Builder for /api/products
│       ├── Item/
│       │   └── WithIdItemRequestBuilder.cs  # Builder for /api/products/{id}
│       └── ProductsResponse.cs           # Response model
└── Models/
    └── Product.cs                        # Generated Product class
```

## 📦 Files Generated by Kiota

### File: `Models/Product.cs`

```csharp
namespace ServiceA.Generated.ServiceBClient.Models;

/// <summary>
/// Automatically generated class representing a Product
/// </summary>
public class Product
{
    public int? Id { get; set; }
    public string? Name { get; set; }

```

**Kiota created a C# class with typed properties!**

---

### File: `Api/Products/ProductsRequestBuilder.cs` (simplified)

```csharp
namespace ServiceA.Generated.ServiceBClient.Api.Products;

public class ProductsRequestBuilder
{
    private readonly IRequestAdapter _requestAdapter;
    private readonly string _urlTemplate = "{+baseurl}/api/products";

    public ProductsRequestBuilder(IRequestAdapter requestAdapter)
    {
        _requestAdapter = requestAdapter;
    }

    /// <summary>
    /// GET /api/products
    /// </summary>
    public async Task<List<Product>?> GetAsync()
    {
        var requestInfo = new RequestInformation
        {
            HttpMethod = Method.GET,
            UrlTemplate = _urlTemplate
        };

        return await _requestAdapter.SendAsync<List<Product>>(requestInfo);
    }
}
```

**Kiota created a `GetAsync()` method that:**
- Knows the URL is `/api/products`
- Knows it returns `List<Product>`
- Handles all serialization/deserialization automatically

---

### File: `ApiClient.cs` (simplified)

```csharp
namespace ServiceA.Generated.ServiceBClient;

public class ApiClient
{
    private readonly IRequestAdapter _requestAdapter;

    public ApiClient(IRequestAdapter requestAdapter)
    {
        _requestAdapter = requestAdapter;
        Api = new ApiRequestBuilder(_requestAdapter);
    }

    /// <summary>
    /// Builder for /api
    /// </summary>
    public ApiRequestBuilder Api { get; }
}
```

**This is the main client you use in your code!**

## 🚀 How to Use the Generated Code

### ❌ BEFORE (manual HttpClient - without Kiota)

```csharp
app.MapGet("/api/users/with-products/{id}", async (
    IHttpClientFactory httpClientFactory) =>
{
    var httpClient = httpClientFactory.CreateClient();
    httpClient.BaseAddress = new Uri("http://serviceb");

    // 1. Make manual request
    var response = await httpClient.GetAsync("/api/products");

    // 2. Read content as string
    var json = await response.Content.ReadAsStringAsync();

    // 3. Deserialize manually
    var products = JsonSerializer.Deserialize<List<Product>>(json);

    // ⚠️ Problems:
    // - URL can have a typo: "/api/prodcts" 💥
    // - Deserialization can fail at runtime
    // - No IntelliSense
    // - If API changes, you only find out in production

    return Results.Ok(products);
});
```

---

### ✅ NOW (Kiota - type-safe)

```csharp
app.MapGet("/api/users/with-products-typed/{id}", async (
    ServiceBClientFactory clientFactory) =>
{
    // 1. Create client (with automatic URL discovery)
    var client = await clientFactory.CreateClientAsync();

    // 2. Call API in a type-safe way
    var products = await client.Api.Products.GetAsync();
    //                          ↑        ↑       ↑
    //                       /api   /products  HTTP GET method

    // ✅ Benefits:
    // - Full IntelliSense
    // - If you type it wrong, it won't compile
    // - If API changes, code breaks at compile-time
    // - Automatically generated models

    return Results.Ok(products);
});
```

**See the difference:**

```csharp
// ❌ HttpClient: everything is a string
var response = await httpClient.GetAsync("/api/products"); // can have typo
var json = await response.Content.ReadAsStringAsync();     // manual
var products = JsonSerializer.Deserialize<...>(json);      // can fail

// ✅ Kiota: everything is typed
var products = await client.Api.Products.GetAsync();       // type-safe!
//                          ^^^^^^^^^^^
//                          IntelliSense shows options
```

## 🛡️ Contract Break Prevention

### Scenario: ServiceB changes its API

```diff
// ServiceB changes field "name" → "productName"
{
  "id": 1,
- "name": "Laptop",
+ "productName": "Laptop",
  "price": 999.99
}
```

**Updated OpenAPI:**
```diff
"Product": {
  "properties": {
    "id": { "type": "integer" },
-   "name": { "type": "string" },
+   "productName": { "type": "string" },
    "price": { "type": "number" }
  }
}
```

**ServiceA runs Kiota again:**
```bash
$ kiota generate -d ../ServiceB/openapi.json
```

**Generated code changes:**
```diff
public class Product
{
    public int? Id { get; set; }
-   public string? Name { get; set; }
+   public string? ProductName { get; set; }
    public double? Price { get; set; }
}
```

**ServiceA code no longer compiles:**
```csharp
var products = await client.Api.Products.GetAsync();

foreach (var product in products)
{
    Console.WriteLine(product.Name); // ❌ COMPILATION ERROR!
    //                        ^^^^
    // 'Product' does not contain a definition for 'Name'
}
```

**✅ Problem detected BEFORE deploy!**

The dev needs to update:
```csharp
Console.WriteLine(product.ProductName); // ✅ Fixed
```

## 💡 Final Analogy

**Kiota is like:**

You have a friend (ServiceB) who owns a store. They give you a catalog (openapi.json) describing:
- Which products they sell
- How much they cost
- How to place orders

Kiota reads that catalog and creates a **robot assistant** (generated code) that:
- Knows all the products
- Knows the prices
- Places orders automatically for you
- Notifies you if anything changes in the catalog

**Result:** You don't need to memorize the catalog or place orders manually. Just tell the robot: "bring me the products!" and it does everything. 🤖

## 🎓 Conclusion

**Kiota:**
1. Reads an OpenAPI file (API contract)
2. Automatically generates C# code
3. Creates classes, methods, everything typed
4. You use that code with IntelliSense
5. If the API changes, code won't compile (safety!)

**Benefits:**
- ✅ Zero manual boilerplate
- ✅ Type-safe (compile-time errors)
- ✅ Full IntelliSense
- ✅ Safe refactoring
- ✅ Contracts validated at build-time

**In the POC:**
- ServiceA generates client for ServiceB
- ServiceB generates client for ServiceC
- Both use automatic discovery for URLs
- Result: Type-safe + Zero hardcoding! 🚀
